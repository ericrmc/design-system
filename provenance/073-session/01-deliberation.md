### §3.1 z8 — capture-adapter-contract-before-capture-mechanism (P3-originated)

P3 [`01c, Frame critique + Q1`]: "the portable design surface is better described as a capture-adapter contract with multiple producer implementations. CM1 can be the first adapter for the current Claude Code self-development workspace, but treating it as the preferred end-state risks laundering current-harness familiarity into engine design. The engine's evidence contract should survive migration to codex CLI, Gemini CLI, OpenAI Agents SDK, custom MCP agents, or non-interactive batch harnesses."

**Synthesis disposition**: substantively adopted. The (γ-6) Q1 framing names CM1 (Claude Code hooks) as **first adapter** at S074 phase-3.1 implementation; CM3 (post-hoc reconstruction) as **secondary bridge/comparator** at S074 phase-3.1; CM2 (external wrapper) as **named portability end-state** deferred to post-S076 review per VD-003 gating. The capture-adapter-contract framing is the architectural envelope: schema (§5.4 below) carries `capture_adapter` + `capture_adapter_version` + `capture_capabilities` metadata; future migration from CM1 to CM2 is adapter-substitution rather than schema-rewrite.

Preserved as **§10.4-M32** first-class minority (substantively adopted; preserved against future-arc rollback toward Claude-only capture). [synth] The reframe extends §10.4-M28 P3 measurement-authority-separation reframe (S071) by adding cross-harness-portability dimension to the producer_kind/authority_level distinction.

### §3.2 z9 — validator-as-evidence-consumer-not-live-harness-participant (P3-originated)

Preserved as **§10.4-M33** first-class minority (substantively adopted; preserved against future-arc rollback toward live-substrate-required validation).

### §3.4 z-laundering-1 — measurement-authority-not-inherited-from-YAML-container (P4-originated)

P4 [`01d, Frame critique + Q2`]: "measurement authority is not inherited from the YAML container. Authority attaches to the capture path and field semantics, not to the existence of a digest-shaped file. ... a top-level pair is insufficient if one digest can contain mixed-origin fields. ... a harness-emitted file can contain an agent-declared workaround description, and that text must not inherit primary status from the file-level header. **Synthesis disposition**: substantively adopted. The (γ-6) Q2 schema scope (SCD-3 target with capture-adapter metadata) carries **field-level authority rules**: each schema section/record carries its own `producer_kind` + `authority_level` (not inherited from file-level). Explicit non-promotion rule: `agent-declared` records cannot be promoted to `harness-measured` solely through digest shape; promotion requires explicit harness-emission via CM1/CM2 capture-adapter. The (γ) phase-3 implementation at S074 phase-3.1 spec text (substantive `validation-approach.md` v7 → v8) MUST include the non-promotion rule per §10.4-M28 + this reframe.

Preserved as **§10.4-M34** first-class minority (substantively adopted; preserved against future-arc rollback toward file-level-only authority).
