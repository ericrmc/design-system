## D-057: Adopt synthesis recommendations and produce the external artefact

**Triggers met:** [d016_3, d016_4]

**Triggers rationale:** Reasonable practitioners genuinely disagreed (d016_3) — the four-perspective deliberation surfaced a 2-2 split on voice (first-person plural vs second-person plural), a majority-vs-minority split on whether to produce a protocol at all (three-of-four yes; Skeptic preferred no-artefact), and divergence on move count (3–4 vs 5 vs 5–6). Operator-marked load-bearing (d016_4) because this is an external-artefact production per D-045 Branch B and the session's primary work-product. d023_* not triggered: the decision does not modify `methodology-kernel.md`, does not create or revise any specification in `specifications/`, and does not assert an OI-004 state change. Non-Claude participation was included voluntarily (continuing the Sessions 005–009 conservative-inclusion pattern), and the Outsider's contributions materially shaped four adopted elements of the artefact — see Key arguments carried below.

**Decision.**

1. **Adopt the synthesis's shape recommendation** (§Recommendations 1). Produce a prose-format external artefact approximately one page in length, with five named moves as headed short sections. Sequential backbone with explicit permission to pause between Move 3 (narrowing) and Move 4 (workable yes). No numbered mandatory steps, no flow diagram, no decision tree, no two-column layout, no matrix.

2. **Adopt the synthesis's content recommendation** (§Recommendations 2). The five moves:

   1. **Name the decision.**
   2. **Say what each of us cares about.**
   3. **Narrow to real options.**
   4. **Look for the workable yes.** — Includes the Outsider-originated burden-cost asymmetry clause: *caring more does not automatically beat ongoing inconvenience, cost, or maintenance burden*.
   5. **Close it, or name what's next.**

   Plus an optional one-sentence after-note at the artefact's foot, not numbered as a sixth move. Form honours the 2-2 Drafter+Mediator yes / Outsider+Skeptic no split via minimal presence.

3. **Adopt the synthesis's register recommendation** (§Recommendations 3). Voice: Outsider's hybrid — second-person plural addressing the couple ("you two") with first-person example lines modelling the couple's internal voice ("I care more about the look of this than the price difference"). Plain English verbs. Invitational when possible; short plain imperatives when plain is clearer. No corporate-register-markers (stakeholders, alignment, action items, framework, criteria, decision-maker). No therapy-register-markers (hold space, check in with feelings, honour perspectives). No "partner" as a label. The Skeptic's single verbatim wording for the uneven-caring prompt is adopted: **"who cares more about this one?"** with "this one" doing de-escalation work.

4. **Adopt the synthesis's title recommendation** (§Recommendations 4). Artefact title: **"A House Decision in Five Moves"** (Outsider-proposed). Drafter's **"Deciding Together"** preserved as secondary candidate if the adopted title reads as too declarative in domain validation.

5. **Adopt the synthesis's "Not this" section** (§Recommendations 5). Four items: no scoring matrix; no written case; no third-party call; no cool-off rule. Heading: **"Not this"** (Skeptic's proposal), not "What this refuses" (which has become methodology-vocabulary). Voting thresholds and explicit veto rights are deliberately *not* listed per the Skeptic's meta-refusal argument [01d, Q5] that naming a refusal can draw attention to a possibility the reader should not have considered.

6. **Adopt the scope-refusal note** (§Recommendations 6). The artefact names that it is calibrated for small-to-medium household decisions (the A$500-or-so band) and is not intended for renovations, vehicles, or life decisions. Mediator-sourced (01b, Q6). This is folded into the "Not this" section or into a short framing note.

7. **Adopt the structural requirement** (§Recommendations 7) that moves be individually addressable — each move's text must stand alone enough that a reader arriving mid-artefact without having just read the previous move can still use it. Outsider-originated [01c, Q6].

8. **Produce the artefact** at `applications/010-household-decision-protocol/house-decision-five-moves.md` — the first use-from-scratch of the `applications/NNN-[slug]/` canonical path (per D-054, Session 009). File frontmatter includes `originating_session: 010` and other required fields per `workspace-structure.md` v2. No regularization step needed (the file is produced at the canonical path).

9. **Preserve dissent** per the synthesis's Preserved Minority Positions section: Skeptic's "no protocol at all" null-alternative benchmark; Drafter's "we"-throughout voice fallback; Skeptic's "regret in a year" dedicated-move refinement; Outsider+Skeptic's no-after-note position.

**Substantive-versus-minor classification per OI-002 heuristic.** Not applicable — no specification is being revised. This decision produces an external artefact, which is not a specification.

**Key arguments carried.**

1. **Four-way convergence on the artefact's core content** [01a, 01b, 01c, 01d, all at Q2]: name the decision; name uneven caring; close cleanly. This is the strongest cross-model signal in the deliberation — all four perspectives independently arrived at the same three-move nucleus. Any shape that omitted any of these three would fail to honour the convergence.

2. **Outsider-unique burden-cost asymmetry clause** [01c, Q2]: "Caring more does not trump the other person's cost, work, or ongoing inconvenience." Adopted into Move 4 because without this clause, the "who cares more" move becomes a preference-wins rule, which produces exactly the resentment failure mode the Mediator named ("one party silently stockpiles disagreement"). No Claude perspective produced this clause; it is Outsider-original.

3. **Outsider-originated hybrid voice** [01c, Q3] resolves the 2-2 Claude split on person (Drafter+Skeptic "we" vs Mediator "you two"). The hybrid — second-person addressing with first-person example lines — is the only proposal that accommodates both positions. This is the second session in a row (Session 009 Q2 was the first) where the synthesizer adopted an Outsider-originated third-way to resolve a 2-2 Claude split.

4. **Outsider-originated title** [01c, Q1]: **"A House Decision in Five Moves."** The most specific and least coy of the three proposed titles. Describes rather than inflates; echoes Morning Unfurl's precedent of naming the count without dramatising it.

5. **Outsider-originated asynchronous-use structural requirement** [01c, Q6]: "This has to work asynchronously and in fragments." Concrete structural implication: each move must be self-contained enough to be picked up across messages. No Claude perspective named this; it is Outsider-original and load-bearing for the artefact's prose density.

6. **Mediator's failure-mode attribution** [01b, Q2]: each of the five moves maps to a specific named failure mode (preferences compounding on who-speaks-first; under-statement to be agreeable; surface-criteria masking deeper stakes; decision paralysis; process-drift). This attribution does not appear in the artefact text (which would read as process-documentation) but shapes the design rationale and is preserved in this decision record.

7. **Skeptic's "this one" de-escalator** [01d, Q4]: the wording "who cares more about this one?" with "this one" limiting the framing to the present decision. Adopted verbatim.

8. **Skeptic's meta-refusal argument** [01d, Q5]: the "Not this" section lists the refusals that might sneak back in during revision, not the obvious ones. Voting thresholds and explicit veto rights are omitted from "Not this" because naming them would signal they were considered. Adopted.

9. **Drafter's register discipline** [01a, Q3, Q6]: specific corporate-register-markers ("stakeholders", "alignment", "action items", "framework", "best practices") named as harmful; the word "protocol" flagged as hazardous vocabulary that would drift the artefact corporate. Adopted: the artefact is titled "A House Decision in Five Moves", not any form including "protocol".

10. **Four-way convergence on what the artefact refuses** [01a, 01b, 01c, 01d, all at Q5]: weighted scoring matrices, written justifications, mandatory cool-off periods, third-party arbitration. All four are named in the "Not this" section (with the cool-off phrasing refined per Skeptic's suggestion to not invite attention to decisions the reader wouldn't otherwise make).

**Rejected alternatives (preserved as dissent).**

- **Skeptic: no artefact at all, or the smallest possible object (single index card of 3 prompts).** [01d, Q1]: "These two people have made household decisions together for however long they have been a household. The methodology is now proposing to formalise a process that has been running un-formalised for years." Rejected on three-of-four convergence on producing a 5-move artefact, but preserved as the null-alternative benchmark against which the artefact's domain-validation report should be read. If domain validation reports that the couple found the artefact unnecessary, the Skeptic's position is the explicit warrant for deprecating the artefact or revising the methodology's governance-domain claim.

- **Drafter: "we"/"us"/"our" throughout with no second-person addressing of the couple.** [01a, Q3]: "'We' positions the reader as already inside the couple the document is for; 'you' positions the document as coming from somewhere outside and instructing." Rejected because the 2-2 split was resolved by Outsider's hybrid, but preserved as fallback if the hybrid reads awkwardly in use.

- **Skeptic: "what would we regret in a year?" as a dedicated named move.** [01d, Q2]: "Medium-term satisfaction is the actual success criterion the user named. A regret check is more honest than a benefits check." Rejected as a named move (compressed into Move 2's "what each cares about" territory), but preserved as a testable refinement. If domain validation reports that the couple didn't surface future-regret before deciding, the Skeptic's position is the explicit warrant for adding the move.

- **Outsider + Skeptic: no optional after-note / no post-decision reflection.** [01c, Q2 & Q5]; [01d, Q2 & Q5]. Rejected in favour of the footnote form (not a numbered move), which splits the difference between Drafter+Mediator's qualified yes and Outsider+Skeptic's no. Preserved; if the footnote is ignored in use or reads as institutional, the artefact is revised to remove it.

- **Drafter: section heading "What this isn't".** [01a, Q5]: "The heading 'What this isn't' rather than 'What this refuses' — because 'refuses' has become methodology-vocabulary." The Skeptic proposed **"Not this"** [01d, Q3]: "renamed so it does not read as meta-commentary on the protocol itself." Skeptic's proposal adopted as slightly shorter and more curt; Drafter's "What this isn't" preserved as secondary candidate.

- **Mediator: "bullet point symbols" as harmful register-markers** [01b, Q3]: "Numbered steps are fine; bullets within steps read as slides." The artefact as designed uses headed short sections rather than bullet lists within moves; Mediator's concern is substantively honoured, though the "Not this" section uses dash-bullets rather than numbered list. Partial adoption.

**Non-Claude participation:** Included (Outsider = OpenAI GPT-5 via `codex exec`). Not required per D-023 (no kernel change, no spec change in categories covered by D-023, no OI-004 state change). ## D-058: Record Session 010 observations and watchpoints

**Triggers met:** [d016_3, d016_4]

**Triggers rationale:** Reasonable practitioners genuinely disagreed (d016_3) on whether the methodology's domain-general claim extends to governance-domain artefacts — Skeptic argued explicitly against [01d, Q6]; Mediator flagged as a risk [01b, Q6]; Outsider cautioned ("over-civilized overdesign" [01c, Q6]); Drafter was the sole perspective seeing no substantive concern [01a, Q6]. Operator-marked load-bearing (d016_4) for session-level housekeeping about what Session 010's work-product has taught the methodology. d023_* not triggered.

**Decision.** Record the following five observations/watchpoints for use by Session 011+ and as inputs to any future methodology-applicability deliberation. Watchpoints are not kernel changes; they are surfaced concerns with concrete activation triggers.

**Watchpoint WX-10-1: Methodology-vocabulary leakage into external artefacts.**

- *Finding:* All four perspectives converged [01a 01b 01c 01d, all at Q6] on the risk that the methodology's own register (protocol, framework, specification, kernel, deliberation, domain validation, artefact) would leak into the produced artefact's surface text and drift the artefact toward corporate-process register.
- *Session 010's response:* The artefact is titled "A House Decision in Five Moves", avoids all methodology-vocabulary in its body text, and adopts Outsider's "over-civilized overdesign" diagnosis as an active discipline.
- *Activation trigger for future deliberation:* a domain-validation report on any external artefact reports the text reads as methodology-adjacent or corporate-process-shaped, despite the session's best efforts. That would be evidence that the vocabulary-hygiene discipline is insufficient and a more structural separation is needed (e.g., a dedicated "artefact-voice-review" step, or a non-Claude drafter for external-artefact language).

**Watchpoint WX-10-2: Asynchronous/fragmented-use requirement for external artefacts.**

- *Finding:* Outsider-unique observation [01c, Q6] that real household decisions are conducted across messages, days, and contexts — not in single sit-down conversations. This has a concrete structural implication: external artefacts that assume single-sitting use may read beautifully and fail in practice.
- *Session 010's response:* The five-move artefact is designed so each move's text stands alone; moves are individually addressable; the footnote after-note is explicitly optional.
- *Activation trigger for future deliberation:* a domain-validation report on any external artefact reports that the artefact was tried only in single-sitting mode, or that fragmented use failed. That would be evidence that async-use requirements need explicit artefact-design guidance.

**Watchpoint WX-10-3: Aim-at-obsolescence success criterion for governance-domain artefacts.**

- *Finding:* Skeptic-framed [01d, Q6]: "Good is an artefact the couple tries once, finds mildly useful, and then internalises and no longer needs." For governance-domain artefacts, continued dependence on the artefact may be failure, not success. The success criterion differs from (say) Morning Unfurl, where continued daily use of the sequence is plausibly compatible with success.
- *Session 010's response:* The domain-validation report from the user/wife will be interpreted with this criterion in mind — "we used it once and didn't need it the second time" reads as stronger evidence than "we used it every time."
- *Activation trigger for future deliberation:* the success-criterion question generalises beyond Session 010. Future external artefacts in governance-adjacent domains may need the aim-at-obsolescence criterion specified at design time. This could eventually shape `methodology-kernel.md` §7 Domain validation text (adding a notion of "fit-dependent success criteria" by artefact type). Deferred until more data.

**Watchpoint WX-10-4: Validation-signal under-power for single-couple domain validations.**

- *Finding:* Skeptic-unique [01d, Q6]: "The user is also the person who commissioned the artefact, the person who will read the deliberation output before trying the artefact, and the methodology's primary author. The chain of motivated interpretation is long." Combined with Mediator's observation that the wife's reaction is one person's taste [01b, Q6, echoing 01d, Q6]. The Session 010 validation is under-powered relative to its claimed implications for methodology domain-generality.
- *Session 010's response:* The domain-validation report will be reported with its under-power explicitly flagged; n=2 external validations (Sessions 008, 010) is better than n=1, but neither represents generalisable evidence.
- *Activation trigger for future deliberation:* OI-004 closure criterion 3 (recorded impact) is satisfied structurally by accumulated data points. But criterion-4 articulation should explicitly address the validation-signal robustness question. Deferred until an OI-004 closure deliberation is undertaken.

**Watchpoint WX-10-5: Pattern — synthesizer adopts Outsider-originated third-way to resolve 2-2 Claude splits.**

- *Finding:* Second consecutive session (Session 009 Q2 applications/-yes-or-no; Session 010 Q3 voice) where a 2-2 Claude split was resolved via the synthesizer adopting an Outsider-proposed resolution no Claude perspective produced. Both times, the Outsider's mechanism (copy-plus-reference in 009; hybrid voice in 010) made one or both Claude positions partially satisfiable while introducing a structural innovation.
- *Interpretation:* This is expected behaviour under OI-004 closure criterion 3 — non-Claude participation is meant to produce resolutions the Claude family does not. The pattern suggests the mechanism is working as designed, not that it is drifting.
- *Activation trigger for future deliberation:* if a future 2-2 Claude split is NOT resolved by an Outsider third-way, that would be novel data worth examining. Also: if the synthesizer appears to be *reaching for* the Outsider position when a Claude position would do, that would be the failure mode (synthesizer bias toward cross-model-signal-shaped resolutions). Monitor.

**Substantive-versus-minor classification per OI-002 heuristic.** Not applicable — no specification is being revised. These are watchpoints, not spec changes.

**Key arguments carried.** Each watchpoint's own Finding and Session 010's response sections.

**Rejected alternatives (preserved as dissent).** None. The watchpoint form is a lightweight record; no competing proposals.

**Non-Claude participation:** Included (Outsider). Not required. Two of the five watchpoints (WX-10-2, WX-10-3 in part) are Outsider-sourced or Outsider-amplified.
