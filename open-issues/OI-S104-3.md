# Promote halted to a sessions.status value via calibrated table-rebuild when warranted

S104 deferred the halted-as-status-value admission per DV-S104-7 because sessions table-rebuild is heavyweight; halt is currently encoded via review_passes.outcome=nonconverged.

Methodology specifications/methodology.md describes a halted session as not a normal close and the next session reopens the work. The single-column read property (querying sessions.status alone reveals halt-state) is desirable but requires sessions table-rebuild because SQLite column-level CHECK cannot be widened in-place. Sessions has eight triggers and dozens of FK referrers, making the rebuild non-trivial and risky. S104 ships the lighter encoding (halt is encoded by review_passes.outcome=nonconverged on a session.status=closed row). Promote when (a) a future session genuinely needs the single-column read property, or (b) the calibrated table-rebuild dance can be amortised with another sessions schema change.
