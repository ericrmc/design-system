## Problem statement

The engine's path-selection mechanism is warrant-gated and sparse-by-design per `methodology-kernel.md` §1 Read + `multi-agent-deliberation.md` v4 §Limitations + `read-contract.md` v4 §5.3. The default-agent path recommendation produces Path A (Watch) whenever no activation warrant fires at session open. This is the intended architecture.

Operator observation at Session 043 open: **six of the last ten sessions (S033–S042) have been pure Path A (Watch); default-agent reasoning produces Path A whenever no activation warrant fires at session open; warrant-firing mechanisms are sparse by design; the resulting observational sessions generate metrics (no-growth streaks, preservation depths, zero-event watchpoint windows) that read as positive outcomes and confirm restraint as the right discipline.** The Session 021 Outsider §5.3 "indefinitely movable finish line" minority warned about this dynamic and was strongly vindicated at Session 041 after twenty successor sessions; that vindication should have informed Session 042 path selection but did not.

Perspective 3 (Long-Baseline Auditor) quantified the pattern: Path-A rate is 47% post-engine-v4 (Sessions 024–042) versus 0% pre-engine-v4 (Sessions 001–023). The Path-A concentration is epoch-specific, not a full-history constant.

Four of five perspectives (including the non-Claude lineage) concluded that something forward should change in response to this pattern. The fifth perspective (Path-Selection Defender) concluded the operator's framing is partially correct on facts but wrong on conclusions, and that any of the proposed mechanism changes converts working discipline into laundering surface; that position is preserved as §5.12 first-class minority with operational reopen warrants (see `provenance/043-session-assessment/01-deliberation.md` §5).

**No 3-of-5 convergence on the specific mechanism to address this was reached at Session 043.** Hence this OI.

## Unresolved sub-questions

Each sub-question below carries the proposing perspective and the alternative frame if one was articulated.

(a) **Reframe choice: "work-channel diversity" (P1) or "warrant-surface closure against operator-frame observations" (P4)?**
Both are operationally coherent; neither is falsified by the other. P1 argues the path-selection gating is sound but the engine's *channels* for generating work have quieted simultaneously (OI-advancement post-OI-004; §9 trigger 7; §5-series firings). P4 argues the warrant surface is dense against engine-internal trajectories but sparse against operator-frame observations, and that operator-frame observations route via Case-Steward discretion (Path-PD-class per S036 precedent) rather than spec. These are two different axes of the same asymmetry. Choosing one may foreclose the other; preserving both as parallel frames may be the right disposition.

(b) **Should preserved-minority activation warrants themselves have expiration windows (P3)?**
Per Long-Baseline Auditor: §5.3 fired at Session 027 but conversion-to-spec happened only at Session 028; vindication at Session 041 showed the "indefinitely deferrable" pattern operationally. P3 proposes: when a preserved-minority activation warrant fires, the session should deliberate within three sessions or the minority graduates to "preserved but no longer monitored" status. This would add deadline enforcement to the minority-tracking system.

(c) **Should the engine have state-decay or pressure triggers on closable/articulated OI state after X sessions (P5)?**
Per Gemini: if an OI is in a "Closable" or "Articulated" state for more than X sessions (~6 per rotation window), a Priority-Review warrant should fire, elevating the issue to the agenda. This is different from (b) — (b) concerns minority warrants; (c) concerns OI state. Both would operate on similar mechanics (time-in-state triggers).

(d) **Should the warrant surface be enlarged at spec level with a new "operator-frame observation" input class (P4)?**
Per Outsider: currently the warrant surface accepts operator input as directive (ratification), agenda-surfacer (Path-PD-class), and reference-text-provider (Path-C-class). There is no spec-level category for "the operator has noticed a frame-level property the engine has not represented to itself." Such observations route via Case-Steward discretion, not through a spec-level warrant. The proposal is to add an explicit input class. Risk: this may itself become a laundering surface (operator can generate work at any time by labelling an observation "frame-level").

(e) **Should the default-agent convention (D-129) be codified at spec level with a specific shape?**
D-129 adopts the convention at minor level (convention without spec amendment). The full shape (≥1 vs ≥2 alternatives; short vs long form; Path-A-justification-section vs single bullet) is deferred to S046 evaluation. OI-019 captures the spec-codification question for whichever shape survives the 3-session verification window.

(f) **What extended-baseline visibility mechanism, if any, is warranted?**
Perspectives divided: P3 favors periodic (once per 6-session window, read-only observational, diagnostic not generative); P4 favors distribution-property-triggered ("Path-A > 60% of last 10 sessions without operator-surfaced-agenda"); P5 favors periodic (every 10 sessions, synchronized with §10.4 monitoring); P2 rejects "periodic" but concedes narrower targeted-read-further-back convention if P3 produced evidence (which P3 did). No 3-of-5 convergence on shape. All variants share: read-only observational, diagnostic not generative, anti-laundering guards.

## Provisional closure criteria

These criteria may be refined during OI-019's lifetime. Any refinement requires deliberation.

1. **Operational evidence** from the engine's continued running that one of the proposed mechanisms is load-bearing — i.e., absence of it demonstrably caused a surface-missed observation or drift. "We have a hunch it might help" is not sufficient.

2. **Cross-family deliberation with ≥3 perspectives converging on a specific mechanism shape.** Session 043's 4-of-5 "something should change" conclusion is insufficient; the convergence must be on *the specific thing*.

3. **Anti-laundering adjudication** showing the adopted shape is not retrofit to rationalise past Path-A concentration. The Path-Selection Defender's §5.12 reopen warrant (b) specifically tests this.

4. **§5.12 engagement.** Any OI-019 deliberation MUST address the §5.12 operational reopen warrants (a/b/c) by argument; failure to address them is itself a laundering signal.
