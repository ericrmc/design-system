### OI-081-001: Should the table-count budget (078 D-10) be replaced by a derived-only check?

**Source.** Minority position from S081 deliberation D-2 (substrate `deliberation_id=2`, sealed `2026-04-27T06:14:18.722Z`), held by `P-2-adversarial` (Anthropic family, separate context, briefed adversarially); recorded as substrate `synthesis_points.synthesis_point_id=5` (kind = 'minority').

## The position

The table-count budget set by 078 D-10 (≤ 16 tables, breachable with cause) is a **derived quantity masquerading as a constraint**. The substrate can compute its own table count via `SELECT COUNT(*) FROM sqlite_master WHERE type='table'`; the validator already runs pre-commit. The budget number is not the protective mechanism — the count is. The protective mechanism that the prose-budget-plus-manifest-enumeration pair carries is *narrative discipline about the count*, and that mechanism produced two miscounts of two different tables across three sessions (079→080→081):

- 079 D-2 acknowledged D-10's enumeration of 16 had inadvertently omitted `synthesis_points`.
- 080 `00-assessment.md` surfaced that `engine-manifest.md`'s prose enumeration sat at 15, omitting both `objects` *and* `synthesis_points`.

This is exactly the failure mode `specifications/constraints.md` property 1 ("models default to prose") says the substrate exists to eliminate, recurring one layer up. The post-restart engine's restraint on accretion is measured in active-spec lines and agent-context load, not in DDL object counts; the table-count budget is a vibe expressed numerically.

## What S081 D-1 did with this position

S081 D-1 ratified 17 as the table count via D-10's existing breach clause (disposition A from the deliberation) and updated the manifest enumeration to align with the schema. This is the disposition the 078 D-5 release gate admits cleanly. Acting on the adversarial position (replacing D-10's budget mechanism — disposition D from the deliberation) would have been a substantive rewrite of an active 078 decision, which the release gate forbids.

## What a post-gate session should consider

After the first 30-session external-problem trial closes and the release gate lifts:

1. Should the table-count budget be removed and replaced by a `selvedge validate --table-budget` derived check (count from `sqlite_master`, refuse pre-commit on count > N where N is set in workspace state, not in spec prose)?
2. Should the manifest's prose enumeration of substrate tables be removed entirely, with the manifest pointing to the schema as the source of truth and to a `selvedge query "SELECT name FROM sqlite_master WHERE type='table' ORDER BY name"` invocation as the canonical listing?
3. Or — the case for keeping D-10 as is — does the human-readable budget number in spec prose serve a different function (orienting new readers, anchoring the subtractor's budget review at human-cadence intervals) that derived checks would not preserve?

The deliberation that takes this up should convene cross-family perspectives (the original deliberation's minority position came from an Anthropic-family adversarial agent; a fresh non-Anthropic perspective on whether the budget *as a mechanism* still earns its keep would be load-bearing).
