### OI-012: `validate.sh` hard-coded `02-decisions.md` path
**Surfaced:** Session 009 (per D-055; the underlying tooling sub-finding was recorded in Session 008 W2 per D-051)
**Status:** Open — defer; Session 009 explicitly declined a tool fix
`tools/validate.sh` checks 14 and 15 hard-code the path `02-decisions.md` as each session's decisions file. Session 008 surfaced that this creates a silent-bypass hazard if a session numbers other files (e.g., artefacts) into positions that shift the decisions file to `03-` or later. Session 008 worked around this by using a non-numbered filename for the external artefact; Session 009 removed the collision source by moving external artefacts to `applications/` (D-054). The hard-coded path therefore no longer has an active collision pressure. **Session 010** was the first use-from-scratch of the `applications/NNN-[slug]/` canonical path; the external artefact is at `applications/010-household-decision-protocol/house-decision-five-moves.md` (outside the provenance directory), and provenance files in `provenance/010-household-decision-protocol/` follow the standard `01-...`, `02-decisions.md`, `03-close.md` numbering without collision. The defect remains available to bite under different future patterns (e.g., a future session wanting variable decisions-file numbering; or a session generating multiple deliberation artefacts that force renumbering). **Re-visit trigger:** a future session needing variable decisions-file numbering, or a second independent concrete collision with the hard-coded path, whichever comes first. The preferred starting point for the future deliberation is the Reviser/Outsider pattern-match proposal (match `[0-9][0-9]-decisions.md` within each session directory; fail if zero or multiple matches) preserved in D-055 Rejected Alternatives.
