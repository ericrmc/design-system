# legacy_imports.decomposition_status does not track actual decomposition coverage

Perspectives 1-5 carry decomposition_status='ratified' but have zero perspective_position/perspective_claim rows; column is wishful.

The legacy_imports.decomposition_status column is set by the import / submit path at write time but is never validated against the actual presence of decomposed rows. Concretely: Perspectives 1-5 (S080 smoke-A/B and S081 claude/codex/adversarial) carry decomposition_status='ratified' but have zero perspective_position rows and zero perspective_claim rows. Their body_md remains the only carrier. The decomposition_status column is wishful — it says ratified when no atomized rows exist. The substrate-canonical contract is broken at exactly the point the column was designed to enforce. What's needed: a derived-or-checked decomposition_status that either (a) computes from actual atom counts at query time and refuses 'ratified' when atom counts are zero; (b) flips automatically when atomization completes; or (c) stays manual but is gated by a CHECK trigger that refuses 'ratified' without at least one decomposed row. Coordinate with OI-085-001 (re-decompose v2 specs) which exercises the same atomize-and-flip pattern on the spec-side.
