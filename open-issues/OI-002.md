# Threshold for substantive revision vs. minor correction

When does a specification edit count as substantive (file-level v-bump) versus minor (no v-bump)?

Decision D-004 established that substantive revisions trigger file-level version preservation while minor corrections do not. No formal criteria for distinguishing the two have been defined. Heuristic across 13 data points (Sessions 002-028): minor if the change makes explicit what the specification's existing language already contains or anticipates as a category of extension, OR annotates existing candidate entries with descriptive metadata within the section's stated purpose; substantive if the change adds new normative content (rules, required fields, severity decisions, gating rules, triggers, required artefacts) beyond what the existing language contains. Pattern observation n=4 narrow-single-purpose-spec creations (identity.md, reference-validation.md, engine-manifest.md, read-contract.md) suggests a third category distinct from substantive vs minor revision.
